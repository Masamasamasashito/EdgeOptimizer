スキルの開発は従来の「感覚で修正を繰り返す」アプローチから、**「テスト・計測に基づく改善」**や**「効率的なコンテキスト管理」**を重視するフェーズへと大きく進化した。

# 1. スキル作成のコア設計原則

Anthropic公式ガイドラインでは、Claudeの能力を最大限に引き出しつつ、トークン消費や推論の混乱を抑えるための設計が推奨されている

## 簡潔さと段階的開示（Progressive Disclosure）

- コンテキストウィンドウ（AIが一度に処理できる情報量）は有限であるため、メインの指示書である`SKILL.md`は500行以下に抑えることが推奨されている
- 一度に全ての情報を読み込ませるのではなく、概要と条件のみを`SKILL.md`に書き、詳細なAPIリファレンスや実行スクリプト、テンプレート（アセット）は別ファイルに切り出す
- AIが必要と判断したタイミングで追加ファイルを読み込むようにすることで、コンテキストの汚染を防ぐ

## 「Claudeはすでに賢い」という前提に立つ

- AIが既に知っている一般的な知識を長々と説明するのは避け、**そのプロジェクトやドメイン固有の専門知識、独自のルール、具体的な手順**を提供することに注力すべき
- 宣言的な説明よりも、具体的なチェックリストやフォーマットのテンプレートを提示する方が効果的

## 制限と自由度のコントロール（決定論的アプローチ）

- 失敗が許されない重要なタスクや、完全に自動化したい定型作業については、Pythonなどのスクリプトに処理を委ねて実行させる「決定論的」なアプローチが有効
- 一方、AIの推論を活かしたい場合は、要件のみを定義して具体的なアプローチはAIに任せるといったバランス調整が重要

## Descriptionの明確化と用語の一貫性

- スキルが適切なタイミングで呼び出されるかどうかは、設定内の`description`（説明文）に依存
- 対象ファイルやトリガーとなる条件を具体的に記述し、スキル内全体で用語を統一することで、意図しない誤発動や未検知を防ぐ

# 2. 「勘と経験」から「計測と改善」へのパラダイムシフト

公式の`skill-creator`の大幅なアップデートにより、ソフトウェア開発における「テスト」の文化がスキル作成にも持ち込まれた

## Eval（評価テスト）による自動判定

- 見出しの数や出力フォーマットなど、期待する結果を「チェックリスト（合格基準）」として定義し、AIの出力が基準を満たしているかを自動で判定（Eval）できる
- AIモデルがアップデートされた際のスキルの品質劣化をいち早く検知できる

## ベンチマークとA/B比較

- テスト結果（合格率、処理時間、トークン使用量）を継続的に記録し、スキルの改善状況を数値で追跡する「ベンチマーク機能」が追加された
- 旧バージョンと新バージョン、あるいはスキルあり/なしの結果をAIに目隠しで比較させる「A/B比較」により、客観的な品質評価が可能

## Descriptionの自動最適化

- スキルクリエイターがサンプルのプロンプトに対してテストを実行し、スキルの発動精度（誤発動や未検知の削減）を高めるために、現在の`description`を自動的に最適化してくれる

# 3. プロのベストプラクティスを自プロジェクトに導入する

- 自分でスキルを作成するだけでなく、Marketplace（https://skills.shなど）を通じて、VercelやMicrosoft、Googleといったトップ企業やプロのエンジニアが作成した専門知識を「スキル」としてプロジェクトに導入することも重要

## 開発品質の自動チェックと担保

- 「Vercel React Best Practices」などの公式スキルを導入すると、自分の書いたコードが最新のフレームワークのベストプラクティス（不要な再レンダリングの防止やバンドルサイズの最適化など）に準拠しているかをAIが自動でチェックし、修正してくれる

## UI/UXや特定領域の強化

- 「UI UX Pro Max」などのスキルを使えば、参考となるWebサイトのURLを渡すだけで、そのサイトのデザインシステム（タイポグラフィやカラースキーム）を抽出し、自身のプロジェクトに反映させるといった高度な作業が可能

## 運用プロセスの自動化

- コード変更時にドキュメントの更新漏れを防ぐ「update-docs」や、GitHubのPRから「リリースノート」を自動生成するスキルなど、手作業でのミスが起きやすいワークフローをスキルで自動化することが強く推奨

# 最新のベストプラクティス

質の高い専門知識をAIが処理しやすい最小限のコンテキスト（段階的開示）で提供し、テスト機能を用いて客観的に品質を計測・改善していくこと

# 主なスキルの一覧

## 1. フロントエンド・UI/UX設計
1. `vercel-react-best-practices`** (Vercel公式): React/Next.jsの最新ベストプラクティスに準拠しているかをチェックし、不要な再レンダリングの防止やバンドルサイズの最適化（同的インポートなど）を行う。
2. `ui-ux-pro-max`: 参考サイトのURLを渡すだけで、そのサイトのデザインシステム（タイポグラフィ、カラースキームなど）を抽出し、自プロジェクトのUI/UXに反映させる。
3. `frontend-design`: UI/UXのデザイン思考（極限までミニマル、アートデコなど）をAIに学習させ、美的感覚に優れた洗練されたフロントエンドコードを生成する。
4. `next-cache-components` (Vercel公式): Next.jsのキャッシュ機能において、コンポーネント単位でキャッシュ戦略の最適化を行う。
5. `zustand`: 状態管理ライブラリZustandのストア設定やセレクターの最適化を行い、無駄な再レンダリングを防ぐ。
6. `i18n`: Reactの国際化において、翻訳キーをフラット構造で管理し検索性を向上させる。
7. `frontend-patterns`: Reactの各種状態管理パターンを提供する。
8. `web-artifacts-builder`: Reactなどで精巧なHTMLアーティファクトを構築する。

## 2. バックエンド・データベース・API
1. `backend-patterns`**: Node.js/Expressのアーキテクチャやエラーハンドリングのベストプラクティス。
2. `spring-boot-patterns` / `spring-boot-security`**: Java Spring BootのREST API設定や、認証・認可の実装パターン。
3. `golang-patterns`: Go言語らしいコード記法（エラーハンドリングやゴルーチンの使い方）を強制する。
4. `fastapi-templates` / `fastapi-htmx`: Python FastAPIのテンプレート構成や、HTMXと組み合わせる際の注意点。
5. `jpa-patterns`: JPAでのエンティティ設定や、N+1問題を回避するクエリ最適化。
6. `prisma-expert`: Prisma ORMのN+1問題回避や、本番環境でのマイグレーション事故を防ぐ設定。
7. `supabase-postgres-best-practices`: SupabaseのPostgreSQLスキーマ設計やセキュリティ（RLS）設定を最適化。

## 3. テスト・セキュリティ
1. `testing` (ローブハブ)**: Vitestなどでのユニットテストにおいて、実装の詳細に依存しない壊れにくいテスト（スパイの活用など）を実装する。
2. `webapp-testing`** (Anthropic公式): Playwrightを用いて、静的サイトと動的Webアプリを自動判別しE2Eテストを実行する。
3. `auth-implementation-patterns`** (OS インプレメンテーションパターンズ): JWT、OAuth、RBACなどの認証周りの安全な実装パターン。
4. `security-review`**: コードレビュー時に、XSSやCSRFなどの脆弱性やセキュリティホールを事前に検知する。

## 4. 自動化・開発ワークフロー補助
1. `update-docs`** (Vercel公式): コードの変更を検知し、影響するREADME等のドキュメント更新・コミットを自動化する。
2. `release-notes-generation`** (Microsoft公式): GitHubのPRからリリースノート（機能追加、バグ修正など）を自動生成する。
3. `pr-creator`**: リポジトリのPRテンプレートに沿った品質を担保してからPRを作成する。
4. `code-reviewer`**: ローカルの変更に対するセルフレビューの品質を向上させる。
5. `session-logs`**: jqを利用し、AIとの会話のセッションログからエラーなどを効率的に分析する。
6. `dispatching-parallel-agents`**: 独立したタスク（フロントとバックエンドのバグ調査など）を複数のエージェントで並列処理させる。
7. `command-development`** (Anthropic公式): テスト実行やデプロイなどの繰り返し作業を独自のスラッシュコマンド化する。
8. `typescript-review`**: 大規模チーム向けのTypeScriptコードスタイルレビュー基準。
9. `coding-agent`**: AIエージェント間の連携（PTY通信など）を安定して呼び出すための設定。

## 5. 動画・マーケティング・分析
1. `remotion-best-practices`**: コードベースで動画を生成する「Remotion」のベストプラクティス。プロ品質のPR動画などを自動作成する。
2. `cro-methodology`** (CRO メソドロジ): ユーザー心理に基づき、売上向上やコンバージョン率（CV）を最適化するための仮説・施策を提案する。
3. `funnel-analysis`** (自作例): アプリのLP訪問から課金までの離脱率（ファネル）を計算・分析し、ボトルネックを特定する。
4. `seo-audit`（SEO監査）、`copywriting`（コピーライティング）、`marketing-psychology`（マーケティング心理学）など。

## 6. ドキュメント処理・汎用ツール
1. `pdf` / `pdf-processing`**: PDFからのテキスト・表抽出、フォームレイアウト分析、および自動入力ワークフロー。
2. `bigquery-skill`**: BigQueryのスキーマ取得や、財務・製品データなどの分析。
3. `slide-dev`**: Markdownベースで開発者向けのスライドを生成・バージョン管理する。
4. `docx`（Word編集）、`pptx`（パワポ生成）、`xlsx`（Excel分析）、`search`（検索）。

## 7. スキル管理・検索ツール（メタスキル）
1. `skill-creator`** (Anthropic公式): 開発者が自身でスキルを作成・管理するためのツール。チェックリスト形式のテスト（Eval）、A/B比較、ベンチマーク計測、およびDescription（説明文）の自動最適化などが行える。
2. `find-skills`**: MarketPlace上に存在する無数のスキルの中から、自分の課題や要件に最も適したスキルを検索して提案してくれる。
3. `claude-men`**: コンテキストウィンドウの圧迫を防ぐためのコンテキスト圧縮（Context Compression）プラグイン。

## 8. その他、Agent Skills Directory ランキング上位（抜粋）
ディレクトリには膨大な数が存在しますが、トップ層には各社公式機能が多く名を連ねています。
- 1. Microsoft/Azure公式系: `azure-ai`、`azure-observability`、`azure-compute`、`azure-cloud-migrate`、`microsoft-foundry`
- 2. Vercel/Next.js系: `web-design-guidelines`、`vercel-composition-patterns`、`deploy-to-vercel`、`turborepo`
- 3. その他注目機能: `agent-browser` (ブラウザ自動操作)、`ai-image-generation` (画像生成)、`elevenlabs-tts` / `elevenlabs-music` (音声・音楽生成)、`shadcn` (UIコンポーネント)、`browser-use`。
