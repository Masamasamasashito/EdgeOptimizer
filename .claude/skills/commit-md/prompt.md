# commit-md スキル

ステージ済みの変更を分析し、わかりやすいコミットメッセージを生成する。

## 作業手順

1. `git status` でステージ済みのファイルを確認
2. `git diff --staged` で各ファイルの差分を取得
3. 差分内容を分析:
   - 何を変更したか（機能追加、バグ修正、ドキュメント更新など）
   - なぜ変更したか（意図や背景）
   - どの範囲に影響があるか
4. コミットメッセージを生成:
   - **絵文字プレフィックス**で開始（例：`:sparkles:`, `:recycle:`, `:docs:`, `:update:`）
   - 冒頭 1 行は簡潔に（50 文字以内を推奨）
   - 必要に応じて詳細は改行後に記述
5. **ユーザーにコミットメッセージを提示して確認を取得**
6. 承認後、`git commit` を実行
7. 承認後、`git push` を実行

## 絵文字プレフィックスの使い分け

- `:sparkles:` - 新機能追加
- `:recycle:` - リファクタリング、コード改善
- `:docs:` - ドキュメント更新
- `:update:` - 一般更新、設定変更
- `:bug:` - バグ修正
- `:arrow_up:` - 依存関係更新

## 重要

- コミットメッセージは**必ずユーザーに確認**してから実行
- 既存のコミットスタイルを維持する
- 変更内容が複雑な場合は、分割コミットを提案する
