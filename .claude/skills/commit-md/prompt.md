# commit-md スキル

ステージ済みの変更を分析し、Edge Optimizer プロジェクトの規約に準拠したコミットメッセージを生成する。

## 基本原則

1. **CLAUDE.md の Git Operation Policy に準拠** — 原則として git コマンドは人間が実行
2. **ユーザーに選択肢を提示** — 実行許可、メッセージ修正、手動実行のいずれかを選択可能
3. **Co-Authored-By ヘッダーの自動追加** — すべてのコミットメッセージに署名行を含める

## 作業手順

### 1. ステージ済み変更の分析

```
git status        # ステージ済みのファイルを確認
git diff --staged # 差分内容を取得
```

### 2. 変更内容の分類

差分を以下の観点で分析：
- **変更の種類**: 機能追加、バグ修正、リファクタリング、ドキュメント、設定変更
- **変更の範囲**: どのファイルが影響を受けるか
- **変更の意図**: なぜこの変更が必要か

### 3. 絵文字プレフィックスの選択

プロジェクトのコミット履歴に基づいた絵文字を使用：

| 絵文字 | 種類 | 使用例 |
|--------|------|--------|
| `:sparkles:` | 新機能 | `:sparkles: 新機能の追加` |
| `:docs:` | ドキュメント | `:docs: README の更新` |
| `:update:` | 一般更新 | `:update: 設定ファイルの修正` |
| `:recycle:` | リファクタリング | `:recycle: コードのリファクタリング` |
| `:bug:` | バグ修正 | `:bug: 認証ロジックの修正` |
| `:arrow_up:` | 依存関係 | `:arrow_up: パッケージのアップデート` |
| `:create:` | 新規作成 | `:create: 新しいワークフローの追加` |
| `:feat:` | 機能追加（英語表記） | `:feat: Edge Analyzer の追加` |

### 4. コミットメッセージの生成

以下のフォーマットで生成：

```
<絵文字>: <簡潔な変更内容>

<詳細（必要の場合）>

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
```

**ルール**:
- 冒頭 1 行は 50 文字以内
- 絵文字と本文の間にはスペースを 1 文字
- 変更内容が複雑な場合は改行して詳細を記述
- 最後の行に Co-Authored-By ヘッダーを必ず追加

### 5. ユーザーへの確認（必須）

**AskUserQuestion ツールを使用して以下の選択肢を提示**：

質問：「コミットメッセージを生成しました。どのように進めますか？」

選択肢：
1. **コミットを実行する** — git commit を AI が実行
2. **メッセージを修正して実行** — 入力欄で修正後、AI が実行
3. **手動で実行する** — コマンドのみ提示して終了

### 6. 選択に応じた動作

**選択肢 1: コミットを実行する**
- `git commit` を実行してコミットを作成
- 必要に応じて `git push` を提案

**選択肢 2: メッセージを修正して実行**
- 入力欄から修正内容を取得
- 修正後のメッセージで `git commit` を実行

**選択肢 3: 手動で実行する**（CLAUDE.md 準拠）
- 以下の情報のみを提示して終了：
  - コミット対象ファイルの一覧
  - コミットメッセージ（コピー＆ペースト可能形式）
  - 実行コマンドの例

## 出力例

### 手動実行の場合（選択肢 3）

```
## コミット準備情報

### 対象ファイル
- Local_LLM_Tuning/Claude_ollama_commands.md

### コミットメッセージ
:docs: ローカル Ollama 起動コマンドの修正

  PowerShell スクリプト記法の修正

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>

### 実行コマンド
git commit -m "$(cat <<'EOF'
:docs: ローカル Ollama 起動コマンドの修正

  PowerShell スクリプト記法の修正

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
EOF
)"
```

## 重要事項

- **必ずユーザー確認を取得** — 無断でのコミット実行は禁止
- **CLAUDE.md を参照** — プロジェクトの Git Operation Policy に従う
- **変更が複雑な場合は分割コミットを提案** — 1 コミットに多数の変更を詰め込まない
- **ファイルパスはフルパスで記載** — リポジトリルートからの相対パスを使用
